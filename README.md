## Calculator

This project is a browser-based calculator built as part of the Foundations section of The Odin Project. The goal was to recreate the core functionality of a standard calculator using HTML, CSS, and JavaScript.

The application supports basic arithmetic operations, button-based input, and display updates using DOM manipulation. The focus of the project was implementing the calculator logic, handling user interactions through event listeners, and managing edge cases such as repeated operators and clearing values.

This project helped reinforce core programming concepts including functions, event handling, state management, and incremental problem solving.

## Lessons Learned

#### 1. Start with Structure Before Coding
At first the calculator project felt overwhelming, so I began by writing pseudocode and a rough structure. Breaking the problem into smaller pieces made it easier to understand what components I needed to build.

#### 2. Functionality Before Appearance
I intentionally moved quickly through the HTML and CSS. My goal was to create something that resembled a calculator and worked, rather than focusing on making the interface perfect. This allowed me to focus on the core JavaScript logic first.

#### 3. Build the Smallest Working Output
When I got stuck, I focused on producing the smallest possible output. For example:
get one button working
get one event listener working
print a value to the display
Once something worked, it became easier to expand the functionality.
This approach helped me regain momentum and stay in flow.

#### 4. Reuse Patterns Once They Work
After implementing the first click event listener, I realized I could reuse the same pattern for other buttons. This made it much faster to build the rest of the calculator functionality.

#### 5. Edge Cases Take Time
The main calculator logic wasn’t the hardest part. Most of my time was spent handling edge cases, such as:
- unexpected inputs
- repeated operators
- clearing values
- maintaining correct calculator state

Working through these issues helped me understand how small details affect program behaviour.

#### 6. Incremental Progress Leads to Flow
Once the basic calculator functions were working, adding additional features became easier. Each small improvement helped maintain momentum and allowed me to stay focused for longer coding sessions.

#### 7. Ownership of the Solution
I wrote most of the logic myself and only searched for syntax or small implementation details. This made the project feel like something I genuinely built, which was motivating and reinforced what I had learned.
